打造自己专属的思维脚手架

曾经看费曼访谈中有一段对话非常能引发人思考：

1. 每个人的思考方式都有所不同（比如有些人是视觉优先，有些人是听觉优先[^1]）
2. 不写就无法思考
3. 我无法构建的，就是我不懂的

所以，人和人的思维方式差异是如此之大，每个人都应寻找自己专属的笔记方法，而没必要强求自己学习某个既定的模式。

所以，先从哪个工具入手不重要，先要记起来。要拿起就能记录自己的思考的，就是最趁手的工具

当你使用一段时间后，自然会迭代出自己的个性化需求：

- 我需要Markdown
- 我需要wikilink
- 我需要……

于是你会开始寻找尝试各种笔记软件、解决方案

但其实每个人都有CODE[^2]的需求

当下这个网络环境，无论是写博客、微博，还是公众号，都可以实现分发/分享的诉求。

到这个阶段，你会觉得哪个工具都不能完美解决自己的问题。

所以，学习适合自己的需求的JavaScript技术栈是必要的。

如果你觉得会一点HTML + CSS + JavaScript，能够写一些简单的页面，或是使用DeepSeek生成，就能够满足你的需要，那OK，就很好，0基础大概一周时间，也就能写一个简单页面了。

但学习使用思维脚手架也是会上瘾的。

于是，对思维脚手架使用的正反馈会让你上瘾。你会不断去学习新的技术，去完善你大脑🧠中追求的“完美范式”
